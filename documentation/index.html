.htaccess

C'est un fichier de configuration du serveur Web Apache. Placé à la racine, les configurations qu'il contient s'appliquent au répertoire en cours et à tous ceux qu'il contient.

Ce fichier contient les instructions pour l'URL Rewriting.

index.php

Ce fichier comporte la structure générale de l'application. Il s'occupe d'afficher le contenu de la bonne page ou encore d'afficher l'entête et le pied de page

conn_bdd.php

Fichier comportant les données concernant la connexion à la base de données. Il Contient également l'instanciation de l'objet PDO

fonctions.php

Ce fichier comporte toutes les fonctions et méthodes de l'application

Nom Description
connexion($a, $b, $location) Cette méthode est appelée en ajax. Elle retourne un message indiquant si les identifiants sont corrects ou pas. Si les identifiants sont corrects, la personne en est informée et est redirigée sur sa page en cours.
importe_ajax_function($id_page,$xajax) Cette méthode est appelée à chaque chargement d'une page afin d'importer les fonctions xajax nécessaires pour une page donnée.
afficher($id_page) Méthode qui en fonction de l'identifiant de la page, passé en paramètres affiche le contenu de la page.
charger($a) Méthode qui permet d'avoir une liste des suggestions des personnes en ajax. Elle prend en paramètre le début du nom qui a été saisi et renvoie une liste des couples "Nom Prénom".
resumer($id) Cette méthode prend en paramètre l'identifiant de la personne et retourne en ajax un résumé sous forme des tableaux des prêts la concernant.
saisiePret($id_personne="", $materiel, $lic_duree, $lic_version, $marque, $type, $date_circulation, $garantie, $nom_licence_duree, $debut_licence, $duree_licence, $nom_licence_version, $lic_duree_associe, $lic_version_associe, $nom_licence_duree_associe, $debut_licence_associe, $duree_licence_associe, $nom_licence_version_associe, $nvl_marque) Méthode utilisée en ajax pour valider et enregistrer un prêt.
supp_materiel($id_personne,$id_materiel) Méthode utilisée en ajax. Elle prend en paramètre les identifiants du matériel et de la personne pour effectuer une suppression du prêt.
supp_lic_duree($id_personne, $id_lic_duree, $msg, $appel_depuis_ajax) Méthode utilisée également en ajax, permet de supprimer une licence de durée. Elle peut être appelée depuis la methode supp_materiel() pour supprimer les licences de durée associées à ce matériel, dans quel cas le paramètre $appel_depuis_ajax sera égale à false.
supp_lic_version($id_personne, $id_lic_version, $msg,$appel_depuis_ajax) Méthode utilisée également en ajax, permet de supprimer une licence de version. Elle peut être appelée depuis la methode supp_materiel() pour supprimer les licences de version associées à ce matériel, dans quel cas le paramètre $appel_depuis_ajax sera égale à false.
nvo_utilisateur($type_utilisateur, $nom, $prenom, $mail, $login_personne="", $motdepasse="", $mdp_confirm="") Cette méthode est utilisée en ajax. Elle permet de vérifier et enregistrer un nouvel utilisateur
changer_mdp($ancien_mdp, $nvo_mdp, $confirm_nvo_mdp) Méthode appelée en ajax. Elle permet de vérifier et midifier le mot de passe.
deconnexion() Méthode appellée en ajax, permet de détruir la session en cours donc de déconnecter la personne.
modifier_licence($pers_concerne, $type_licence, $id_licence, $nom_licence, $date_debut)
frm_modif_licence($type_licence, $id_licence)
recuperation($mail) Méthode appelée en ajax, permet de vérifier si le mail passé en paramètres existe dans la base de données, générer un nouveau mot de passe et l'envoyer par mail.
genere_mdp() Méthode qui génére et retourne un mot de passe. Elle est appelée à partir de la methode recuperation()
envoyer_mail($destinataire, $login, $mdp) Méthode qui envoit un mail à la personne qui souhaite récupèrer son mot de passe. Elle est appelée à partir de la méthode recupèration().

Détails des méthodes :

connexion

Fonction qui permet l'identification des utilisateurs

Paramètres :
  • $a : Le login entré par l'utilisateur
  • $b : Le mot de passe entré par l'utilisateur
  • $location : La page sur laquelle le formulaire est affiché (pour rediriger l'utilisateur après la connexion)

Retourne : Un message indiquant si la personne est identifiée ou pas. Si la personne est trouvée dans la base elle est redirigée sur la page demandée

importe_ajax_function

Fonction qui permet de déclarer les fonctions ajax en fonction de la page visitée

Paramètres :
  • $id_page : Identifiant de la page en cours
  • $xajax : L'objet xajax sur lequel on effectue les actions (ajout de fonctions)

afficher

Affiche le contenu de la page demandée

Paramètres :
  • $id_page : Identifiant de la page dont on souhaite afficher le cotenu

charger

Afficher une liste des suggestions de personnes

Paramètres :
  • $a : La chaine de caractères entrée par l'utilisateur

Retourne : Liste des personnes(html)

resumer

Afficher un résumé sur les matériels et les licences empruntés

Paramètres :
  • $id : L'identifiant de la personne

Retourne : Les tableaux récapitulatifs (html)

saisiePret

Enregistrer un prêt de matériel ou de licence (les licences peuvent être associées au matériels)

Paramètres :
  • $id_personne : Identifiant de la personne à qui on effectue un prêt
  • $materiel : Est-ce un prêt de matériel (VRAI/FAUX)
  • $lic_duree : Est-ce un prêt de licence de durée (VRAI/FAUX)
  • $lic_version : Est-ce un prêt de licence de version (VRAI/FAUX)
  • $marque : Numéro de la marque (indice dans la liste déroulante)
  • $type : Numéro du type de matériel (indice dans liste déroulante)
  • $date_circulation : Date de mise en circulation du matérel
  • $garantie : Nombre d'années de garantie du matériel
  • $nom_licence_duree : Nom de la licence de durée
  • $debut_licence : Date de début de la licence de durée
  • $duree_licence : Durée de la licence (en années)
  • $nom_licence_version : Nom de la licence de version
  • $lic_duree_associe : Est-ce qu'il y a une association d'une licence de durée (VRAI/FAUX)
  • $lic_version_associe : Est-ce qu'il y a une association d'une licence de version (VRAI/FAUX)
  • $nom_licence_duree_associe : Nom de la licence de durée associée au matériel
  • $debut_licence_associe : La date de début de la licence de durée associée au matériel
  • $duree_licence_associe : Durée de la licence de durée associée au matériel
  • $nom_licence_version_associe : Nom de la licence de version associée au matériel
  • $nvl_marque : Le nom de la nouvelle marque (s'il y en a une)

Retourne : Un message informant de la réussite ou de l'echec de l'enregistrement

supp_materiel

Supprimer un matériel prêté

Paramètres :
  • $id_personne : Identifiant de la personne concernée
  • $id_materiel : Identifiant du matériel à supprimer

Retourne : Message informant de la réussite ou de l'echec

supp_lic_duree

Supprimer une licence de durée

Paramètres :
  • $id_personne : Identifiant de la personne concernée
  • $id_lic_duree : Identifiant de la licence de durée à supprimer
  • $msg : Ce paramètre est utile seulement si la fonction est appelée à partir de supp_materiel(). C'est les messages concernant la réussite ou l'échec de la suppression du matériel associé
  • $appel_depuis_ajax : Faux si la fonction est appellée à partir de supp_materiel() et Vrai dans les autres cas. Sert à déterminer s'il faut effectuer une réponse en ajax ou pas

Retourne : Message informant de la réussite ou de l'echec

supp_lic_version

Supprimer une licence de version

Paramètres :
  • $id_personne : Identifiant de la personne concernée
  • $id_lic_version : Identifiant de la licence de version à supprimer
  • $msg : Ce paramètre est utile seulement si la fonction est appelée à partir de supp_materiel(). C'est les messages concernant la réussite ou l'échec de la suppression du matériel associé
  • $appel_depuis_ajax : Faux si la fonction est appelée à partir de supp_materiel() et Vrai dans les autres cas. Sert à déterminer s'il faut effectuer une réponse en ajax ou pas

Retourne : Message informant de la réussite ou de l'echec

nvo_utilisateur

Créer un nouvel utilisateur

Paramètres :
  • $type_utilisateur : Le type du nouvel utilisateur
  • $nom : Le nom du nouvel utilisateur
  • $prenom : Le prénom du nouvel utilisateur
  • $mail : Le mail du nouvel utilisateur
  • $login_personne : Le login du nouvel utilisateur
  • $motdepasse : Le mot de passe du nouvel utilisateur
  • $mdp_confirm : La confirmation du mot de passe du nouvel utilisateur

Retourne : Message indiquant la réussite ou l'échec de la création du nouvel utilisateur

changer_mdp

Modifier le mot de passe du compte connecté actuellement

Paramètres :
  • $ancien_mdp : L'ancien mot de passe de l'utilisateur
  • $nvo_mdp : Le nouveau mot de passe de l'utilisateur
  • $confirm_nvo_mdp : La confirmation du nouveau mot de passe

Retourne : Message indiquant la réussite ou l'échec du changement de mot de passe

deconnexion

Destruction de la session de l'utilisateur

Retourne : Redirection sur la page d'accueil

modifier_licence

Modifier les données de d'une licence

Paramètres :
  • $pers_concerne :
  • $type_licence :
  • $id_licence : Identifiant de la licence
  • $nom_licence : Le nom de la licence
  • $date_debut : La date du début de la licence

Retourne : Message indiquant la réussite ou l'échec de la mise à jour de la licence

frm_modif_licence

Afficher un formulair prérempli pour modifier une licence

Paramètres :
  • $type_licence : Le type de la licence qu'on souhaite modifier (0:durée 1:version)
  • $id_licence : Identifiant de la licence qu'on veut modifier

Retourne : Formulaire pré rempli

récuperation

Méthode qui permet de reconnaitre l'utilisateur à partir de son email, de générer un nouveau mot de passe et l'envoyer par mail à l'utilisateur

Paramètres :
  • $mail : Le mail de la personne qui souhaite retrouver son mot de passe
Retourne :

genere_mdp

Générer un nouveau mot de passe

Retourne : Nouveau mot de passe généré

envoyer_mail

Envoyer un mail

Paramètres :
  • $destinataire : L'email de la personne à qui envoyer le mail
  • $login : Le login de la personne à qui on envoi l'email
  • $mdp : Le nouveau mot de passe

Retourne : Vrai si le mail a bien été envoyé et Faux dans le cas contrair

jquery.js

Une bibliothèque JavaScript qui a pour but de simplifier les commandes communes de JavaScript.

script_calendrier.js

Script pour le calendrier. Facilite la saisie des dates dans les champs spécifiques.

calendrier.css

Les styles pour le calendrier.

styles.css

Fichier comportant tous les styles du site web

styles-mobile.css

Fichier de styles pour les appareils mobiles

404.php

Fichier qui contient le contenu de la page d'erreur 404, c'est-à-dire un message indiquant que la page est introuvable

accueil.php

Fichier qui contient le contenu de la page d'accueil, c'est-à-dire quelque données statistiques sous forme d'un tableau de bord

administration.php

Fichier qui comporte le contenu de la page d'administration, c'est-à-dire le formulaire de création d'un nouvel utilisateur

entete.php

Fichier qui comporte l'entête du site : le logo, le menu, le nom de l'utilisateur connecté et le bouton de déconnexion

form_connexion.php

Fichier qui comporte le formulaire de connexion en html

form_recuperation.php

Fichier qui comporte le formulaire de récupération du mot de passe en html

pied.php

Fichier qui comporte le pied de page en html

profil.php

Fichier qui comporte le contenu de la page profil, c'est-à-dire le formulaire pour changer le mot de passe

resume_prets.php

Fichier qui comporte la structure de la page récapitulative des prêts, notamment le champ de saisie du nom de l'utilisateur

saisie_prets.php

Fichier qui contient le formulaire de saisie d'un prêt

xAJAX
